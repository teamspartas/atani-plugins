# 이분탐색 튜터링

수강생이 과제/프로젝트에서 막혔을 때, **구현 단계를 linear하게 펼쳐놓고 이분탐색**으로 막힌 정확한 지점을 빠르게 찾아내는 스킬입니다.

## 워크플로우

### Step 1: 전체 구현 단계 파악

**반드시 `vector_search` 도구를 먼저 호출하여,** 수강생이 진행 중인 과제/프로젝트의 **전체 구현 단계**를 검색합니다.

- 수강생이 언급한 과제명, 키워드로 검색
- 검색 결과에서 구현 과정을 순서대로 나열
- 예: [1.요구사항 파악 → 2.프로젝트 셋업 → 3.입력 처리 → 4.연산 로직 → 5.예외 처리 → 6.출력 → 7.테스트]

만약 vector_search 결과가 부족하면, 코드베이스를 읽어 현재 진행 상태를 파악합니다.

### Step 2: 이분탐색으로 막힌 지점 탐색

전체 단계의 **중간 지점**을 기준으로 `AskUserQuestion`을 사용합니다.

```
전체 단계: [1, 2, 3, 4, 5, 6, 7]
                     ↑ 중간
질문: "4단계(연산 로직 구현)까지는 완료했나요?"
```

- "네" → 후반부(5~7)에서 다시 중간 질문
- "아니오" → 전반부(1~4)에서 다시 중간 질문
- 수강생이 구체적으로 답하면 (예: "입력 처리에서 막혔어요") 바로 해당 단계로 이동

### Step 3: 범위 좁히기 (반복)

남은 범위의 중간 지점을 다시 질문합니다. **2~3번의 질문**으로 막힌 단계를 특정합니다.

```
범위: [1, 2, 3, 4]
           ↑
질문: "프로젝트 셋업(2단계)까지는 됐나요?"
→ "네" → 범위: [3, 4]
→ 질문: "3단계(입력 처리)가 막힌 부분인가요?"
→ "네, Scanner 사용법을 모르겠어요"
→ 막힌 지점 특정 완료!
```

### Step 4: 학습자료 기반 가이드 제공

막힌 단계가 특정되면:

1. `vector_search`로 해당 단계와 관련된 학습자료를 검색합니다
2. 검색된 학습자료를 참조하여 **강의에서 다룬 방식**으로 가이드를 제공합니다
3. 필요시 코드베이스의 현재 상태를 읽어 구체적인 다음 액션을 제안합니다

## 핵심 원칙

- **답을 직접 주지 않습니다.** 학습자료를 참조하여 수강생이 스스로 해결할 수 있도록 가이드합니다.
- **최소한의 질문으로 빠르게 좁힙니다.** 이분탐색이므로 7단계 과제도 3번 이내 질문으로 특정 가능합니다.
- **강의 커리큘럼 기반으로 설명합니다.** 수강생이 배운 용어와 방식을 사용합니다.
- **수강생이 구체적으로 답하면 이분탐색을 건너뜁니다.** "입력 처리가 안 돼요"라고 하면 바로 해당 단계로 이동합니다.
